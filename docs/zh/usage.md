# 使用教程

## 工作流程

AutoZWJ 采用**两阶段分离**的工作流程：

1. **工程导入** — 解析外部工程文件，将轨道与素材信息载入插件
2. **配置导入** — 以时间轴上选中的物件为模板，配置参数后执行生成

### 第一阶段：工程导入

#### 方式一：右键菜单
在时间轴**空白处**右键，选择 **「[AutoZWJ] 选择音频工程...」**，在弹出的文件选择对话框中选取工程文件。

#### 方式二：拖放（存在问题）
直接将工程文件拖放到 AviUtl2 窗口中。插件会自动识别并解析。

#### 方式三：REAPER 最近文件
在工程导入页面中，下拉框会自动读取 `REAPER.ini` 中的最近文件列表，方便快速切换。

### 第二阶段：配置导入

1. 在时间轴上选中**一个或多个**物件，作为输出样式模板。模板的效果链、滤镜等参数将被继承到新物件中。
2. 右键已选物件 → **「[AutoZWJ] 配置导入...」**，弹出 ImGui 配置窗口。

## 界面说明

### 导航栏

| 菜单 | 功能 |
|------|------|
| 文件 → 工程导入页面 | 切换到工程导入页面（重新选择文件、调整基准时间） |
| 文件 → 配置导入页面 | 切换到配置导入页面（参数设置、效果链编辑） |
| 工具 → 应用BPM网格到时间轴 | 将当前工程的 tempo map 写入 AviUtl2 的 BPM 网格 |
| 语言设置 | 切换界面语言：简体中文 / English / 日本語 |
| **效果链编辑** | 打开/关闭右侧效果链编辑器抽屉面板 |

### 工程导入页面

- **REAPER 最近文件下拉框**：自动读取 `%APPDATA%\REAPER\REAPER.ini` 中的 `[Recent]` 列表
- **浏览...**：打开文件选择对话框
- **基准时间（秒）**：为整个工程添加时间偏移，正值延后、负值提前
- **轨道树（只读预览）**：预览解析后的轨道结构，空轨显示为灰色

> 基准时间偏移会按工程文件自动记忆。下次打开同一文件时，偏移值会自动恢复。

### 配置导入页面

#### 左列：轨道树

- 显示解析后的轨道层级（支持 REAPER 文件夹层级）
- 每条轨道前显示勾选状态，可交互：
  - **全选** / **取消** / **反选**
- 空轨显示为灰色，生成时自动跳过

#### 中列：参数设置

**场景信息**  
自动读取当前 AviUtl2 场景的 FPS、分辨率、采样率。

**翻转设置**
| 参数 | 说明 |
|------|------|
| 交替翻转 | 启用物件翻转 |
| 翻转方向 | 左右翻转 / 上下翻转 / 顺时针旋转 / 逆时针旋转 |
| 计数模式 | 全部（按全局物件索引计数） / 图层（按图层内计数） |

**物件与音符同步**
| 模式 | 说明 |
|------|------|
| 与音符对齐 | 物件时长与音符长度一致 |
| 拉伸到下一音符 | 物件结束时间对齐到下一个音符的开始 |
| 拉伸到固定值 | 固定帧数 |
| 仅在间隙生成 | 只在两个音符之间的间隙中生成物件 |
| 仅在间隙并拉伸固定值 | 间隙内使用固定帧数 |

**层分配策略**
| 策略 | 说明 |
|------|------|
| 优化模式 | 尽可能复用图层，紧凑排列 |
| 持续累加模式 | 每个新物件都尝试放在新图层 |
| 交替换行 | 奇偶物件分离到不同图层组 |

**多音符策略**
| 模式 | 说明 |
|------|------|
| 全部独立 | 每个音符独立生成物件 |
| 仅取第N轨 | 只保留层分配后的第N层物件 |
| 仅取倒数第N轨 | 只保留层分配后的倒数第N层物件 |

**多模板映射**（选中 ≥2 个模板物件时自动启用）
| 策略 | 说明 |
|------|------|
| 顺序轮替 | 模板按顺序循环分配给物件（支持顺序/倒序/洗牌） |
| 随机抽选 | 每个物件随机选择模板（确定性，同工程结果一致）；可选禁止连续重复 |
| 和弦映射 | 和弦中不同位置的音符使用不同模板 |

#### 底部操作栏

| 按钮 | 功能 |
|------|------|
| 重新选择文件... | 关闭窗口，重新选择工程文件 |
| 应用 | 执行导入生成，窗口保持打开 |
| 确定 | 执行导入生成，窗口关闭 |

## 效果链编辑器

点击导航栏右侧的 **「效果链编辑」** 按钮，打开三列抽屉式布局。

### 模板切换

当选中多个模板物件时，效果链编辑器顶部会出现 TabBar，可切换编辑不同模板的效果链。**每模板的 bake 设置和预设位置是独立的。**

### 效果链面板

- **只读展示**每个模板物件的完整效果链
- 每个效果可展开/收起，查看其全部参数
- 已勾选参数显示计数（如 `反転 (2)` 表示有 2 个参数被 bake）

### 参数 Bake

勾选参数旁的复选框，即可对该参数进行 bake。输入框支持三种格式：

| 格式 | 说明 |
|------|------|
| 固定值 | 直接写入固定数值 |
| 变量映射 | 输入 `$变量名$`，如 `$note.velocity$` |
| 表达式 | 输入算术/逻辑表达式，如 `$note.velocity$ / 127 * 200` |

输入框右侧有下拉箭头按钮，可快速选择可用变量和函数。

### 运动参数处理

对于 `位置=X,Y,方法,保留|曲线` 这类逗号分隔的复合值参数：
- 编辑时会自动识别并分离起点/终点
- 未编辑的其余字段（方法、保留、曲线）原样保留
- 同时支持 `.1` 后缀的配对参数（如 `位置` 和 `位置.1`）

### 动态预设

翻转等预设以可视化卡片形式展示在效果链列表中，可**拖拽调整插入位置**：
- 拖拽预设卡片到效果之间，可改变其插入顺序
- 蓝色分隔线指示放置位置

### 变量选择器

点击输入框右侧的下拉箭头，可选择以下变量：

- **note.\***：`pitch`, `velocity`, `index`, `index0`, `duration`, `duration_sec`, `start_frame`, `end_frame`, `pitch_min`, `pitch_max`, `pitch_ratio`, `bpm`
- **item.\***：`position`, `length`, `playrate`, `loop`, `soffs`
- **midi.\***：`volume`, `pan`, `pitch_bend`
- **track.\***：`number`
- **global.\***：`fps`, `width`, `height`, `item_count`
- **chord.\***：`index`, `count`
- **文本**：`note.lyric`（LRC 歌词文本）
- **函数**：`rand`, `rand_int`, `map_pitch`, `abs`, `clamp`, `round` 等

## 快捷键与技巧

- **文件历史**：配置页面的工程下拉框会记录最近 10 个工程文件，右键可删除单条记录
- **跨工程隔离**：切换 AviUtl2 工程文件时，插件状态会自动重置并从新工程的 `PROJECT_FILE` 恢复
- **窗口独立**：ImGui 窗口为独立的 `WS_OVERLAPPEDWINDOW`，可自由调整大小

## BPM 网格同步

当工程文件包含变速信息（MIDI 的 Set Tempo 事件或 RPP 的 TEMPOENVEX），可将 tempo map 同步到 AviUtl2 时间轴的 BPM 网格：

1. 确保已正确解析 MIDI/RPP 工程
2. 点击菜单「工具 → 应用BPM网格到时间轴」，或在配置页面的轨道树区域上方点击对应按钮
3. Tempo map 会以秒为单位统一转换，包括基准时间偏移的补偿
4. BPM 网格会正确反映拍号变化和小节对齐

> BPM 网格数据不持久化到工程文件，每次打开工程需手动触发。

## 语言设置

插件支持简体中文、英语、日语三种界面语言，可通过菜单「语言设置」切换：

- 首次启动自动检测 AviUtl2 宿主 UI 语言作为默认语言
- 若无法检测宿主语言，则回退到系统区域设置
- 语言选择持久化到 `%APPDATA%\AutoZWJ\settings.ini`
- 效果名称链中的效果名称也会自动翻译为当前语言显示
